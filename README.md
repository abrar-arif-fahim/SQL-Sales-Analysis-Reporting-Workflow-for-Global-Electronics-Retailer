# SQL Sales Analysis & Reporting Workflow for Global Electronics Retailer

## Project Overview

This SQL portfolio project demonstrates an end-to-end data analysis workflow for a global electronics retailer. The project uses MySQL to load raw CSV files, validate data quality, clean and transform fields, create reusable reporting views, and answer business questions with SQL.

The project is designed to be beginner-friendly, professional, and GitHub-ready. It does not include hard-coded business results; all outputs should be generated by running the SQL scripts against the database.

## Project Objective

The objective is to turn raw retail CSV files into reliable SQL reporting assets that can support business analysis across sales, products, customers, stores, countries, categories, and delivery performance.

## Dataset

Raw CSV files are stored in `data/raw`:

- `Customers.csv`
- `Products.csv`
- `Sales.csv`
- `Stores.csv`
- `Exchange_Rates.csv`
- `Data_Dictionary.csv`

The dataset includes customer demographics, product details, sales order lines, store locations, exchange rates, and field definitions.

## Workflow

1. **Create tables** using clean column names, appropriate data types, primary keys, indexes, and comments.
2. **Load data** from CSV files into MySQL using `LOAD DATA LOCAL INFILE`.
3. **Run quality checks** for row counts, duplicates, missing values, invalid values, date issues, and relationship gaps.
4. **Create cleaned views** with standardized text, delivery status, delivery days, revenue, cost, profit, and profit margin.
5. **Create reporting views** for reusable sales, product, country, store, category, monthly, and customer analysis.
6. **Run business queries** to answer portfolio-ready business questions.

## How To Run

Create or confirm the database:

```sql
CREATE DATABASE IF NOT EXISTS electronics_retailer;
```

From the project root, connect to MySQL with local infile enabled:

```bash
mysql --local-infile=1 -u root -p electronics_retailer
```

Then run the SQL scripts in this order:

```sql
SOURCE sql/01_create_tables.sql;
SOURCE sql/02_load_data.sql;
SOURCE sql/03_data_quality_checks.sql;
SOURCE sql/04_cleaning_and_transformation.sql;
SOURCE sql/05_reporting_views.sql;
SOURCE sql/06_business_analysis_queries.sql;
```

If MySQL blocks local file loading, enable it:

```sql
SET GLOBAL local_infile = 1;
```
